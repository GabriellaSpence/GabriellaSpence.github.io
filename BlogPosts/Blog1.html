Building a Website

HTML & CSS & JavaScript, oh my!

I have been inspired to build a personal website for a while now, but the process always seemed overwhelming. One day I finally gained the momentum I needed and ended up with a functional website. If you are looking for a practical way to build a website with no knowledge of HTML, CSS, or JavaScript, look no further! Get ready to roll your sleeves, grab some coffee, beseech the Google gods, and get to work.

Step 1: Find an HTML Template

Did you know you don't need to write your whole website from scratch? What a game-changer for me. Some lovely people release their templates for free and allow you to modify them for personal or commercial use. So take some time to Google HTML templates and pick a design that fits your needs. Note: make sure to check the licensing details before you modify.

Need a place to start? I ended up grabbing a template from Free HTML5. If you are looking for other options, I found this website as well TemplateMo.

Step 2: Host the website on GitHub

You can choose to do this step now or at the end. If you want to document all critical changes quickly, I would add your website to GitHub to have version control. However, if you are worried about having your site visible before it is presentable, then do this step at the end.

You will need to have access to a GitHub account. For those of you who have never used GitHub, I find the easiest way to get started is downloading GitHub Desktop, which will allow you to bypass learning all those Git commands. Here is a a resource to get you set up.

Once you have an account and GitHub Desktop ready, you need to make a public repository. Your repository needs to be public to host your website unless you have a Pro account, and the repository name should be "username.github.io." You will now see your repository in your Documents > GitHub folder in your File Explorer. Take your HTML template, unzip, go one level into the folder (you should see the index.html), and copy/paste all those files into your GitHub repository folder.

Step 3: Set-Up Text Editor

First, you need a place to edit your code and view your website. I decided to do this in Visual Studio Code. VS Code is a text editor that works with most languages and (from what I've read) is the top pick for web development. Next, you will need to download the Live Server. extension from the marketplace. This will allow you to open your website files in the browser and automatically update when you save changes to those files.

If you skipped step 2, you will take your downloaded HTML template, make sure to unzip it, rename it if preferred, and save the folder in a location you can find later. Then in VS Code, navigate to File > Open Folder and select the name of your folder. Your setup should look something like this:
