Building a Website

HTML & CSS & JavaScript, Oh My!

I was inspired to build a personal website for a while, but the process always seemed overwhelming. One day I finally gained the momentum I needed and ended up with a functional website. If you are looking for a practical way to build a website without knowledge of HTML, CSS, or JavaScript, look no further! Get ready to roll your sleeves, grab some coffee, beseech the Google gods, and get to work.

Step 1: Find an HTML Template

Did you know you don't need to write your whole website from scratch? What a game-changer for me. Some lovely people release their templates for free and allow you to modify them for personal or commercial use. So take some time to Google HTML templates and pick a design that fits your needs. Note: make sure to check the licensing details before you modify.

Need a place to start? I ended up grabbing a template from Free HTML5. I also found this website, TemplateMo.

Step 2: Host the Website on GitHub

You can choose to do this step now or at the end. If you want to document all critical changes quickly, I would add your website to GitHub to have version control. However, if you are worried about having your site visible before it is presentable, then do this at the end.

You will need to have access to a GitHub account. For those who have never used GitHub, I find the easiest way to get started is by downloading GitHub Desktop, which will allow you to bypass learning all those Git commands. Here is a resource to get you set up.

Once you have an account and GitHub Desktop ready, you must make a public repository. Your repository needs to be public to host your website unless you have a Pro account, and the repository name should be "username.github.io". You will now see your repository in your Documents > GitHub folder in your File Explorer. Take your HTML template, unzip, go one level into the folder (you should see the index.html), and copy/paste all those files into your GitHub repository folder. You new changes should show up in your GitHub Desktop. Create a commit message and push your changes. Wait a few minutes and then you can visit "https://username.github.io/" to view your hosted website.

Step 3: Set-Up Text Editor

First, you need a place to edit your code and view your website. I decided to do this in Visual Studio Code. VS Code is a text editor that works with most languages and (from what I've read) is the top pick for web development. Next, you will need to download the Live Server extension from the marketplace. This extension allows you to open your website files in the browser and automatically update them when you save changes to those files.

If you skipped step 2, you will take your downloaded HTML template, make sure to unzip it, rename it if preferred, and save the folder in a location you can find later. Then, in VS Code, navigate to File > Open Folder and select your folder name. Your setup should look something like this:

Your folder with all the files related to your website will exist in the left panel. Right-click on your index.html and "Open with Live Server." This action will open the website in your default browser.

Step 4: Modify

You will need to invest some time learning the basics. The amount of time you spend depends on what you intend to do with your website and how much you plan to modify it. Do you want to change the content and call it a day? Check out the basics of HTML. Do you want to change colors and formatting? Learn some CSS basics. Do you want to react to user events? Dive into JavaScript. Googling the basics of each will result in a plethora of resources, so I won't attempt to add any here.
